# 面向复杂约束的机组排班问题混合启发式求解框架

本仓库是队伍提交的机组排班项目代码。项目围绕 7 天滚动计划期下的机长排班问题展开，目标是在满足地点衔接、时间衔接、飞行时长、值勤时长、休息时间、资质匹配、地勤任务等复杂约束的前提下，尽可能提高航班覆盖率和机组利用效率。

本项目采用“反向推演 + 混合贪心 + 两阶段抢任务 + ALNS”的启发式优化框架，兼顾初始解质量、局部重构能力与全局迭代优化能力。

## 项目概览

### 问题目标

- 输入数据包含航班、机组、资质匹配、大巴置位、可过夜机场、地勤任务等信息。
- 输出结果为计划期内每位机长的任务排班表，以及用于提交和分析的结果文件。
- 优化方向主要包括：
  - 提升航班覆盖率
  - 提升值勤日日均飞时
  - 减少新增过夜站点、外站过夜天数、置位次数与违规情况

### 算法框架

根据答辩材料，本项目的核心求解流程如下：

1. 反向推演
   从计划期末反向构造理论上的优质终局状态，推断更合理的机组初始分布。
2. 正向混合贪心构造初始解
   以航班驱动整体流程，在微观分配时引入以机组为核心的多维评估机制。
3. 中间启发式改进
   针对任务量较低的机组回收并重建任务链，改善低密度、低效率排班。
4. 两阶段抢任务优化
   在小规模机组群内重构任务分配，平衡工作负荷并提升目标值。
5. ALNS 全局优化
   结合多种破坏与修复算子，并配合模拟退火式接受准则做迭代改进。

答辩材料中的实验结果显示，4 组数据集的航班覆盖率分别达到 87.8%、92.2%、97.0%、97.8%，平均运行时长约 55 分钟。

## 输入数据

程序默认读取以下 6 个 CSV 文件：

- `crewLegMatch.csv`
- `flight.csv`
- `busInfo.csv`
- `crew.csv`
- `layoverStation.csv`
- `groundDuty.csv`

这些文件由主程序在运行时通过相对路径读取。如果你直接在 `Code/` 目录运行脚本，那么上述 6 个 CSV 文件也应放在 `Code/` 目录下；如果你从仓库根目录运行 `python Code/提交代码703.py`，则应把这 6 个 CSV 文件放在仓库根目录，或者自行修改 `main()` 中的文件路径。

## 脚本与数据集对应关系

`Code/` 目录中 4 个脚本的差异主要在于计划期时间参数已经分别针对不同数据集做了预配置。关键变量位于各脚本顶部：

- `PROACTIVE_POSITIONING_START_DATE`
- `OPTIMIZATION_START_DATE`
- `OPTIMIZATION_END_DATE`

当前预置情况如下：

| 脚本 | 大巴置位开始时间 | 计划期开始时间 | 计划期结束时间 |
| --- | --- | --- | --- |
| `提交代码606.py` | 2025-05-26 00:00 | 2025-05-29 00:00 | 2025-06-04 23:59:59 |
| `提交代码623.py` | 2025-04-29 00:00 | 2025-05-01 00:00 | 2025-05-07 23:59:59 |
| `提交代码703.py` | 2024-12-30 00:00 | 2025-01-01 00:00 | 2025-01-07 23:59:59 |
| `提交代码711.py` | 2025-01-30 00:00 | 2025-02-02 00:00 | 2025-02-08 23:59:59 |

如果你更换了数据集，或数据日期范围与上述配置不一致，需要手动修改这些时间参数。

修改原则如下：

- `OPTIMIZATION_START_DATE`：`flight.csv` 中最早日期
- `OPTIMIZATION_END_DATE`：`flight.csv` 中最晚日期
- `PROACTIVE_POSITIONING_START_DATE`：`busInfo.csv` 中最早出现的日期

## 运行方式

### 方式一：进入 `Code/` 目录后运行

```bash
cd Code
python 提交代码703.py
```

这是与原始运行说明最一致的方式。此时输入文件和输出文件都位于 `Code/` 目录下。

### 方式二：在仓库根目录运行

```bash
python Code/提交代码703.py
```

此时脚本仍会按相对路径读取文件，因此默认会从仓库根目录查找输入 CSV，并把输出文件写到仓库根目录。

## 输出结果

程序运行完成后，会生成以下文件：

- `crew_schedule_optimized.xlsx`
  最详细的排班结果
- `rosterResult.csv`
  符合提交格式的核心结果文件
- `crew_summary_optimized.xlsx`
  机组每日摘要
- `violations_summary.xlsx`
  违规详情报告

脚本中的保存逻辑已经内置在主流程里，无需额外调用。

## 提交前注意事项

根据 `运行配置说明.docx`，最终生成的 `rosterResult.csv` 可能包含飞行任务、置位任务和地勤任务。若按比赛提交样例要求不展示地勤任务，则需要在提交前手动删除 `rosterResult.csv` 中的地勤任务行。
